# LandingPreset

Шаблон приложения для быстрой разработки AI-based лендингов на Next.js. 
Используемые технологии:
- Next.js (SSR) + Typescript
- CSS-modules
- Yarn4 (node_modules, без PNP)

## Начальная генерация
Для быстрого создания лендинга нам необходим доступ к одной из передовых LLM-моделей (например - Deepseek).
Базовый промпт для настройки нейросети на работу:

```
Представь, что ты вайбкодер. У тебя есть на работе инструмент, который из JSON создает лендинги. Правила составления контракта JSON следующие:
1) Структура JSON-файла неизменна.
2) Существуют две основные секции: globalCssTokens (которая хранит базовые CSS переменные) и sections, которая хранит описание (верстку и контент) секций лендинга
3) В globalCssTokens есть три суб-блока - base (переменные применяемые независимо от темы), light и dark (соответственно, переменные светлой и темной темы)
4) В sections структура следующая: каждый суб-блок = отдельная секция, название которой совпадает с ключом суб-блока.
5) Внутри каждой отдельной секции обязательно должны быть два суб-блока: styles и mainContent. 
6) В блоке styles хранятся объекты по следующему правилу: ключ = название элемента для библиотеки styled-components. Внутри объекта по ключу - вложенные объекты со следующими ключами: __html - название тэга, в который надо парсить данный элемент, base - стандартное описание CSS свойств, mobile - описание CSS-свойств для адаптивного режима (768px), которые превратятся в блок @media после препроцессинга.
7) В блоке mainContent описывается древовидная структура элементов. Разрешено использовать ТОЛЬКО элементы, которые ты ранее описал в блоке styles и комбинировать их вложенность. Вложенность указывает взаимное расположение элементов и их отношение (родитель-ребенок).
8) Если элемент должен иметь в качестве children текстовый контент, а не другой элемент, то ему указывается поле content с содержимым тега.
9) В mainContent на одном уровне вложенности нельзя использовать элементы с одинаковыми ключами, т.к. ввиду особенностей JSON возьмется только последний элемент.

ВАЖНЫЕ НЮАНСЫ: для styles не поддерживаются анимации. Для элементов JSX не поддерживаются массивы, каждый элемент необходимо описывать отдельно (к примеру: CardOne, CardTwo), соответственно, каждый из элементов массива так же должен быть описан в секции styles. То есть, если нам нужны повторяющиеся по визуалу и положению карточки, мы ОБЯЗАНЫ описать их в блоке styles как одинаковые CardOne, CardTwo и т.д. и соответственно, в блоке mainContent в нужном месте описывать содержимое ИМЕННО КАК CardOne, CardTwo и т.д.

Пример валидного JSON: 

<сюда вставить контракт из settings>
```

После ответа на данное сообщение можно начинать генерировать лендинги. Пример запроса на генерацию:
```
Давай напишем для начала красивый современный лаконичный в приятных цветах лендинг для фитнес-студии. Блоки оформить согласно всем правилам построения CEO-Friendly лендинга.
```

Ответ от нейросети необходимо положить в `src/settings/main_settings.json`, после чего выполняем следующие действия:
1) Устанавливаем необходимые зависимости: `yarn install`
2) Запускаем проект: `yarn dev` 
После первого запуска он может упасть с ошибкой, ничего страшного.
3) В соседнем терминале в корне проекта выполняем: `yarn aigenerate:start`
4) Проверяем результат


## Возможные ошибки и тонкости работы 

Ошибок быть не должно, но иногда нейросеть может не сразу "понять правила", поэтому возможны корректировочные запросы.
К примеру, если нейросеть продолжает ошибаться с элементами в массивах, можно уточнить так:
```
Почему в блоке Hero ты не соблюдал правило работы с массивами ?
Ты создал в секции styles элемент StatItem, а в mainContent использова StatItemOne и StatItemTwo
Так как это первая твоя ошибка, я не лишаю тебя премии, а объясняю, как делать правильно.
Есть два варианта - либо ты создаешь один styles элемент и переиспользуешь его (обязательно с тем же названием StatItem) в mainContent несколько раз, либо ты создаешь несколько элементов в блоке styles с одинаковым содержимым (блоки StatItemOne и StatItemTwo, которые идентичны)
Предпочтительнее - первый вариант.
Исправь в конфиге эту ошибку и аналогичные во всех местах, где они есть
```

Далее, работаем с нейросетью. Если ты сгенерировал новый конфиг, просто положи его в `settings` и выполни команду:
- `yarn aigenerate:reload`
Она автоматически очистит старые сгенерированные файлы и создаст новый лендинг
