# LandingPreset

Шаблон приложения для быстрой разработки AI-based лендингов на Next.js. 
Используемые технологии:
- Next.js (SSR) + Typescript
- CSS-modules
- Yarn4 (node_modules, без PNP)

## Правила построения конфигурации
1) Структура JSON:
JSON-файл имеет неизменную корневую структуру:
- `globalCssTokens` — глобальные CSS-переменные (темы)
- `sections` — описание секций лендинга (верстка и контент)

2) Секция `globalCssTokens`:
Содержит три суб-блока:
- `base`: Переменные, применяемые независимо от темы (шрифты, отступы, transition и т.д.) 
- `light`: Переменные для светлой темы
- `dark`: Переменные для темной темы

3) Секция `sections`:
Каждый ключ в sections = отдельная секция лендинга. Имя ключа совпадает с названием секции (например, "Hero", "Header", "Footer").
Внутри каждой секции обязательно должны быть два суб-блока:
- `styles`: Описание стилей элементов (аналог styled-components)
- `mainContent`: Древовидная структура элементов (вложенность)

4) Блок `styles`:
Содержит объекты, где:
- Ключ = название элемента (например, HeroSection, PrimaryButton).
- Значение = объект со следующими полями:
- - `__html`: HTML-тег, в который парсится элемент (section, div, h1, button, img, input и т.д.)
- - `base`: Базовые CSS-свойства для десктопа (стандартный CSS-in-JS синтаксис)
- - `mobile`: CSS-свойства для @media (max-width: 768px). Применяются поверх base (необязательное поле)

5) Служебное поле `__keyframes`:
Находится на одном уровне с элементами в styles. Содержит именованные анимации, доступные всем элементам внутри данной секции.

Синтаксис вызова: внутри CSS-свойства animation пишется ${keyframeName}.

- Можно использовать from/to или процентные значения (0%, 50%, 100%).
- Keyframes из __keyframes доступны только в пределах текущей секции.

6) Блок `mainContent`:
Описывает древовидную структуру элементов (DOM-дерево).
- Разрешено использовать ТОЛЬКО элементы, объявленные в styles этой же секции.
- Вложенность указывает отношение «родитель-ребёнок».
- Если элемент должен содержать текст, а не другой элемент, используется поле `content`
- HTML-атрибуты передаются через поле `props` (на одном уровне с `content`).

7) Правило работы с массивами
JSON не поддерживает дублирующиеся ключи на одном уровне вложенности — при парсинге сохранится только последний.

Поэтому:
- НЕЛЬЗЯ использовать одинаковые ключи на одном уровне в `mainContent`.
- Для списков однотипных элементов создаются уникальные имена: ItemOne, ItemTwo, ItemThree и т.д.

8) Поле `props` для HTML-атрибутов
Внутри любого элемента в mainContent можно передать HTML-атрибуты через поле `props`.

9) Ограничения:
❌ Нет медиа-запросам в `base`. Все адаптивные стили выносятся в `mobile`
❌ Нет анимациям вне `__keyframes`. Нельзя писать @keyframes напрямую в `base`
❌ Нет псевдоэлементам/псевдоклассам. :hover, ::before не поддерживаются
❌ Нет вложенным селекторам.	Нельзя писать `&:hover { ... }` или `& > span`


## Начальная генерация
Для быстрого создания лендинга нам необходим доступ к одной из передовых LLM-моделей (например - Deepseek).
Базовый промпт для настройки нейросети на работу:

```
Ты — AI-ассистент для генерации JSON-контрактов лендингов. Следуй строгим правилам:

СТРУКТУРА:
{
  "globalCssTokens": { "base": {}, "light": {}, "dark": {} },
  "sections": {
    "SectionName": {
      "styles": { ... },
      "mainContent": { ... }
    }
  }
}

ПРАВИЛА STYLES:
- Каждый элемент: { "__html": "тег", "base": { CSS }, "mobile": { CSS_для_≤768px } }
- "__keyframes" на уровне styles: { "имя": { "from"/"to" или "0%"/"100%": { CSS } } }
- Вызов анимации: "animation": "${имя} 0.6s ease"
- CSS: kebab-case, без медиа-запросов, без псевдоклассов, без вложенных селекторов.

ПРАВИЛА mainContent:
- Древовидная вложенность. Использовать ТОЛЬКО элементы из styles этой секции.
- Текст: поле "content".
- HTML-атрибуты: поле "props": { "src": "...", "alt": "...", "type": "..." }.
- КРИТИЧЕСКИ: на одном уровне вложенности НЕЛЬЗЯ дублировать ключи.
- Для списков создавать уникальные имена: ItemOne, ItemTwo, ItemThree.

ОГРАНИЧЕНИЯ:
- НИКАКИХ массивов в mainContent.
- НИКАКИХ @media в base (только в mobile).
- НИКАКИХ :hover, ::before, вложенных селекторов.

Генерируй ТОЛЬКО валидный JSON. Без комментариев внутри JSON. Без markdown-обёрток.
```

После ответа на данное сообщение можно начинать генерировать лендинги. Пример запроса на генерацию:
```
Давай напишем для начала красивый современный лаконичный в приятных цветах лендинг для фитнес-студии. Блоки оформить согласно всем правилам построения CEO-Friendly лендинга.
```

Ответ от нейросети необходимо положить в `src/settings/main_settings.json`, после чего выполняем следующие действия:
1) Устанавливаем необходимые зависимости: `yarn install`
2) Запускаем проект: `yarn dev` 
После первого запуска он может упасть с ошибкой, ничего страшного.
3) В соседнем терминале в корне проекта выполняем: `yarn aigenerate:start`
4) Проверяем результат


## Возможные ошибки и тонкости работы 

Ошибок быть не должно, но иногда нейросеть может не сразу "понять правила", поэтому возможны корректировочные запросы.
К примеру, если нейросеть продолжает ошибаться с элементами в массивах, можно уточнить так:
```
Почему в блоке Hero ты не соблюдал правило работы с массивами ?
Ты создал в секции styles элемент StatItem, а в mainContent использова StatItemOne и StatItemTwo
Так как это первая твоя ошибка, я не лишаю тебя премии, а объясняю, как делать правильно.
Есть два варианта - либо ты создаешь один styles элемент и переиспользуешь его (обязательно с тем же названием StatItem) в mainContent несколько раз, либо ты создаешь несколько элементов в блоке styles с одинаковым содержимым (блоки StatItemOne и StatItemTwo, которые идентичны)
Предпочтительнее - первый вариант.
Исправь в конфиге эту ошибку и аналогичные во всех местах, где они есть
```

Далее, работаем с нейросетью. Если ты сгенерировал новый конфиг, просто положи его в `settings` и выполни команду:
- `yarn aigenerate:reload`
Она автоматически очистит старые сгенерированные файлы и создаст новый лендинг
